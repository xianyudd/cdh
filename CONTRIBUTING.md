# Commit message convention

```text
<type>(<scope>): <summary>
```

## Types

- `feat`, `fix`, `docs`, `refactor`, `test`, `chore`, `ci`, `perf`

## Preferred scopes

- `install`, `bash`, `fish`, `zsh`, `history`, `paths`, `recommend`, `controller`, `readme`, `release`, `ci`, `tui`

## Rules

- 默认使用中文；如果英文更准确或更自然，可以使用英文
- `summary` 保持简洁、动作导向
- 每条提交聚焦一个主要变更
- 只有在 scope 能增加信息时才填写
